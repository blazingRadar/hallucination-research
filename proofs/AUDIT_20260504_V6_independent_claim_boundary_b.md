# V6 Claim-Boundary Audit Public Note

The original version of this memo mixed pre-run claim-boundary analysis with
non-technical planning. That planning material has been removed from the public
repository surface.

Retained technical conclusion:

- V6 needed to report the preregistered effect-size threshold exactly.
- V6 could not claim production-schema evidence.
- V6 could not isolate field vocabulary from slot count.
- Provider asymmetry needed to be reported as a limitation, not hidden inside a
  pooled result.
- Null, threshold-miss, and high-baseline outcomes all had to be treated as
  informative rather than rescued post hoc.

Current claim boundaries are maintained in `README.md` and
`proofs/AUDIT_SYNTHESIS_V6_POST_RUN_20260504.md`.
